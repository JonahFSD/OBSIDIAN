# AI hard drive 

Curated knowledge base for an AI coding agent. **Start here** — this maps every note; each is read
on demand. Keep notes lean; edit them freely.

  types, don't trust agreement). The Claude Code entry point.
- [[Spec-Driven Build Loop]] — the process spine that sequences the notes below: idea → spec →
  build-to-green → gate → distill. Plan in Cowork, build in Claude Code, distill lessons back here.
- [[Deterministic Gates]] — the tooling that **enforces** the rules (pre-commit, CI, agent hooks, permissions); wire it into a repo.
- [[Deterministic Methodology]] — the named methodologies behind the gates (CbC, DbC, neurosymbolic, constrained decoding, flow engineering); vocabulary to search prior art.
- [[Good Code in the AI Age]] — the durable LLM failure modes that make "good code" good, plus the
  caveats. The "why" behind everything else here.
- [[Invariants]] — the coding invariants that follow from those failure modes.
- [[Code Conventions]] — conventions hub: Conventional Commits and per-language standards.
- [[Writing Tests]] — heuristics for writing good tests.
- [[Writing Documentation]] — heuristics for writing good documentation.
- [[TypeScript Development]] — TypeScript patterns, generics, utility types, type guards, React/Node, `tsconfig`, strict mode.
